# Contributing to sbom-vm

Thanks for your interest in contributing! Please read the repository README first for usage and development notes.

Guidelines
- Open a GitHub issue for significant changes before implementing.
- Keep changes small and focused; one concern per PR.
- Run existing tests and linters if present before submitting a PR.
- Include a clear description and rationale in your PR body.

Code of Conduct
Please be respectful and inclusive in all interactions. Report issues to maintainers if needed.
