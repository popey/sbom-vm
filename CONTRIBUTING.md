Thank you for contributing to sbom-vm!

Getting started
- Read README.md to understand the project's purpose and usage.
- Run the scripts locally: python3 generate-test-images.py and python3 sbom-vm.py <image>

Code style
- Keep changes small and focused. Follow existing Python style.
- Run simple syntax checks: python3 -m py_compile sbom-vm.py

Testing
- There are currently no required unit tests. If you add behavior changes, include tests under the `tests/` directory.

Submitting changes
1. Fork the repository and create a branch for your change (feature/fix). 
2. Commit with a clear message and include a short description in the PR body.
3. Open a pull request against main and describe the motivation, changes, and any testing steps.

Need help?
- Open an issue or reply to an existing one. Maintainers are happy to help guide contributions.

Thanks for helping improve sbom-vm!
